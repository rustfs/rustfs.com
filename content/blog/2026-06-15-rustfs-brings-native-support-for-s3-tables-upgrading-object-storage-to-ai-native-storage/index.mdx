> **Product status update:** RustFS and its S3 Tables work are in public beta and preview respectively. This post describes a direction under active development, not a production-ready feature announcement. Review the current [feature status](https://github.com/rustfs/rustfs#feature--status) and validate the S3 operations your workload requires.

RustFS is exploring an S3 Tables surface for teams that want table-oriented data workflows close to S3-compatible object storage. The goal is to learn where table metadata, snapshots, and maintenance workflows can reduce the operational gap between object storage and lakehouse tooling.

## Why explore S3 Tables?

AI and analytics platforms often keep unstructured objects, table metadata, catalogs, and processing systems in separate operational domains. S3 Tables and Apache Iceberg-oriented workflows offer a possible way to make that boundary more explicit and easier to automate.

The evaluation questions matter more than the label:

- Which S3 Tables APIs and Iceberg workflows are required by the target engine?
- How are snapshots, manifests, compaction, and cleanup observed and recovered?
- Which identity, encryption, and audit controls apply to table operations?
- How do upgrades and rollback behave with real metadata and object counts?

## Current direction

The RustFS work is focused on an open, inspectable implementation that can evolve with the Apache 2.0 project. The intended areas include:

- table-bucket and metadata workflows;
- snapshot and maintenance operations;
- interoperability experiments with Apache Iceberg-oriented tools;
- explicit identity, transport, audit, and encryption review.

These areas are not all complete. KMS and several distributed features also remain under validation in the broader RustFS beta, so they should not be treated as inherited production guarantees for S3 Tables.

## How to evaluate the preview

1. Use a disposable dataset and a pinned RustFS beta release.
2. Record the exact client, engine, API calls, and table format version used.
3. Exercise create, read, update, maintenance, failure, and cleanup paths.
4. Test restart, upgrade, rollback, and partial-failure behavior.
5. Keep source data and a recovery path outside the preview environment.

Feedback and reproducible compatibility reports are welcome in the [RustFS issue tracker](https://github.com/rustfs/rustfs/issues).

## About RustFS

RustFS is an Apache 2.0 S3-compatible object storage project built in Rust. It is currently in public beta. Start with the [GitHub repository](https://github.com/rustfs/rustfs), the [S3 compatibility overview](https://docs.rustfs.com/features/s3-compatibility/), and the current release notes before planning a deployment.
